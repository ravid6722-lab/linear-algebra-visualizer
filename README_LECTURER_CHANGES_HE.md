# עדכונים שבוצעו בהתאם להערות המרצה

## 1. עריכה אלגברית וגיאומטרית דו־כיוונית

נוספה אפשרות לגרור נקודות צבעוניות בקצות הווקטורים בתצוגות 2D ו־3D. הגרירה מעדכנת מיד את הערכים האלגבריים המתאימים:

- גרירת `u` או `v` מעדכנת את רכיבי הווקטור.
- גרירת `A·e₁`, `A·e₂` וב־3D גם `A·e₃` מעדכנת את עמודות המטריצה.
- בצירוף ליניארי, גרירת הווקטור המתקבל מחשבת מחדש את `α` ו־`β` כאשר הווקטורים בלתי תלויים.
- במצב בסיס, גרירת `w` מחשבת מחדש את וקטור הקואורדינטות שלו בבסיס הנבחר.
- לחיצה על `Shift` בזמן גרירה מצמידה את הערכים למספרים שלמים.

במצב שבו סטודנט עוקב אחר המרצה, העריכה הגיאומטרית נעולה כדי למנוע שינוי מקומי של מצב השיעור.

## 2. ממשק בעברית

הממשק הועבר לכיוון RTL ותורגמו לעברית:

- מסכי הבית, המרצה, ההצטרפות והתרגול.
- כל פקדי המטריצות, הווקטורים וההנפשה.
- חלונית התובנות והניתוחים.
- מאגר שאלות ברירת המחדל, הבוחן החי ומסכי ניהול השאלות.
- הודעות שגיאה מרכזיות בצד הלקוח והשרת.

הסימונים המתמטיים המקובלים, כגון `A`, `u`, `v`, `P_B`, `det(A)` ו־LaTeX, נשארו בסימון המתמטי המקורי.

## 3. ארגון צירופים ליניאריים, מרחב נפרס ובסיס

שלושת הנושאים רוכזו תחת קבוצה מודגשת אחת בבורר הנושאים: **צירופים, פריסה ובסיס**. לכל נושא נשמר מצב תרגול עצמאי כדי לאפשר למידה נפרדת של:

- צירוף ליניארי.
- מרחב נפרס ואי־תלות ליניארית.
- בסיס, קואורדינטות ומעבר בין בסיסים.

## 4. מעבר בסיס באמצעות מטריצת מעבר

במצב בסיס נוסף אזור ייעודי המציג:

- את מטריצת המעבר `P_B=[u v]`.
- את `det(P_B)` ובדיקה האם הבסיס תקין.
- את המטריצה ההפוכה `P_B⁻¹`, כאשר היא קיימת.
- את וקטור הקואורדינטות `[w]_B` ואת הווקטור בבסיס הסטנדרטי `[w]_std`.
- המרה דו־כיוונית לפי:
  - `[w]_std = P_B · [w]_B`
  - `[w]_B = P_B⁻¹ · [w]_std`

אפשר לערוך את הקואורדינטות בכל אחד משני הבסיסים ולראות מיד את השינוי באחר ובגרף.

## 5. מרחבים וקטוריים אבסטרקטיים

הורחב אזור המרחבים האבסטרקטיים עם דוגמאות אינטראקטיביות של:

- פולינומים.
- פונקציות.
- מטריצות כאובייקטים וקטוריים.

נוספו המחשה של חיבור וכפל בסקלר, קואורדינטות ביחס לייצוג מתאים, ובדיקות מושגיות של סגירות, איבר אפס, נגדי חיבורי ודיסטריבוטיביות.

## הפעלה מקומית

### שרת

```bash
cd backend
npm install
npm start
```

### ממשק

בחלון נוסף:

```bash
cd frontend
npm install
npm run dev
```

ברירת המחדל היא שרת ב־`http://localhost:3000` וממשק ב־`http://localhost:5173`.

מסד נתונים אינו נדרש להפעלת חדרים חיים. ללא `DATABASE_URL`, ניהול מאגר השאלות משתמש במאגר מקומי בדפדפן.

## בדיקות שבוצעו

- בניית Production של Vite הושלמה בהצלחה.
- בדיקת תחביר לכל קובצי השרת הושלמה בהצלחה.
- נבדקה המרה מבסיס לא־סטנדרטי לבסיס הסטנדרטי ובחזרה.
- נבדק תרחיש מלא: יצירת חדר, הצטרפות מרצה וסטודנט, סנכרון מצב, פתיחת בוחן, מענה, חשיפת תשובה וסגירת הבוחן.

הערה: סביבת העבודה שבה בוצעו השינויים חוסמת פתיחת כתובות מקומיות מתוך Chromium אוטומטי. לכן לא בוצעה בדיקת צילום מסך אוטומטית, אך בוצעו בדיקות בנייה, תחביר, חישובים ואינטגרציית שרת מלאה.

## סבב תיקונים נוסף — יולי 2026

### תיקון מונחים
- המונח "פרישה" הוחלף ב־"פריסה" בכל הממשק, התובנות ומאגרי השאלות.

### תיקון תרגול בסיסים ב־2D
- תוקנה שגיאת JavaScript שגרמה לקנבס להיעלם במצב "בסיס, קואורדינטות ומעבר בסיס".
- נוספו שני בסיסים מלאים: B={b₁,b₂} ו־C={c₁,c₂}.
- ניתן לגרור על הגרף את b₁, b₂, c₁, c₂ ואת w.

### מעבדת מעבר בין בסיסים
- אפשר להזין ישירות את מטריצות הבסיס P_B ו־P_C.
- אפשר לעבור למצב הזנת מטריצת המעבר T(C←B); בסיס C נגזר אוטומטית כאשר המטריצה הפיכה.
- מוצגות ומחושבות שתי מטריצות המעבר:
  - T(C←B)=P_C⁻¹P_B
  - T(B←C)=P_B⁻¹P_C
- אפשר לערוך את [w]B, את [w]std או את [w]C. כל שינוי מעדכן את שתי מערכות הקואורדינטות ואת אותו הווקטור הגיאומטרי.
- נוספו הסברים מורחבים, בדיקות תקינות בסיס, דטרמיננטות ודוגמאות מוכנות.

### המישור כצירופים ליניאריים
- בתרגילי מטריצות נוסף מתג המציג נקודות סריג כצירופים ליניאריים של A·e₁ ו־A·e₂.
- נקודת דוגמה מודגשת באמצעות 2(A·e₁)+1(A·e₂), והיא נעה יחד עם המישור בזמן שינוי המטריצה או ההנפשה.

### הנפשה ועיצוב
- כפתור ההנפשה, האיפוס ובחירת המהירות הועברו לסרגל שמעל הגרף.
- בזמן ההנפשה מוצגת המטריצה A בתיבה קבועה, בעוד שהמישור נע מן הזהות I אל A.
- הפונטים אוחדו לפונט ממשק אחד; הכותרות הוגדלו והודגשו.
- במסכים רחבים לוח הפרמטרים והגרף נשארים זה לצד זה ובאותו קו. לוח הפרמטרים נגלל בתוך עצמו כדי שהגרף יישאר גלוי.
